Go的安装和配置

Jun 08, 2024

安装

警告

本篇安装过程不建议观看，请使用搜索并移步 Go的多版本管理工具：g

Linux

Debian

默认版本仓库可能比较旧，建议不用。

go 没有官方 apt 仓库，只有手动安装来获取所需版本，但是管理go版本比较麻烦，这里使用 gvm来管理Go版本

注意：gvm使用安装go过程并不简单，且容易出现问题。

gvm源码：https://github.com/moovweb/gvm

安装 gvm

1

sudo apt-get install bison

1

bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)

刷新环境变量，查看gvm版本

1
2

source ~/.bashrc
gvm version

安装go，使用gvm安装go之前还需要安装go

1

sudo apt install golang

gvm安装新版本go还得安装一个低版本go为引导

1

gvm install go1.20.6

1

gvm use go1.20.6

再安装 go较高的版本

1

gvm install go1.22.4

1

gvm use go1.22.4

1

go version

配置

GOPROXY

配置代理

七牛云

1
2

go env -w GO111MODULE=on # 启用模块
go env -w GOPROXY=https://goproxy.cn,direct # 加快go模块下载速度

GOPATH

使用以下命令设置 GOPATH（假设设置为 ~/codespace/goProjects）：

1
2

echo 'export GOPATH=~/codespace/goProjects' >> ~/.bashrc
source ~/.bashrc

在 GOPATH 目录下，通常有三个子目录：

  • src: 存放源代码。
  • pkg: 存放编译后的包。
  • bin: 存放go下载的可执行文件。
1

mkdir -p $GOPATH/src $GOPATH/pkg $GOPATH/bin

以后就可以把代码放在 $GOPATH/src下了。

为了方便使用 Go 下载的包的可执行文件，将 $GOPATH/bin 加入到你的 PATH 中（如 ~/.bashrc 或 ~/.zshrc）：

1
2
3

# 适用于unix系统 bash环境
echo 'export PATH=$PATH:$GOPATH/bin' >> ~/.bashrc
source ~/.bashrc
